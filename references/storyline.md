# SCR Storyline and Page Architecture

## Build the deck-level argument

Use:

- **Situation:** stable context the audience accepts.
- **Complication:** change, tension, gap, risk, or opportunity requiring a decision.
- **Resolution:** choices, priorities, capabilities, and actions supported by evidence.

SCR is an argument, not three decorative labels. Each section must create a reason to read the next.

## Create the page plan

For every page specify:

| Field | Requirement |
|---|---|
| Page role | Situation, Complication, Resolution, bridge, or close |
| Conclusion title | A defensible full-sentence judgment |
| Evidence | Source IDs from the evidence table |
| Visual | Chart, table, matrix, flow, map, or structured text |
| Implication | Why the evidence matters |
| Bridge | What question the next page answers |

## Conclusion-title test

Weak:

- Market overview
- Consumer analysis
- Channel strategy

Strong:

- Market growth is recovering, but value is shifting toward structurally advantaged segments
- Core consumers are expanding while purchase logic becomes more polarized
- Channel expansion without differentiated economics will dilute profit

A strong title can be challenged and defended with the page evidence.

## Evidence hierarchy

Build each content page in this order:

1. Conclusion title
2. Primary evidence
3. Secondary evidence or segmentation
4. Interpretation
5. Business implication or action
6. Source note

Do not place a chart without stating what it proves.

## Page capacity

Prefer one principal question per page. High density comes from structured evidence, not tiny text.

Typical page components:

- one conclusion title;
- one dominant chart or analytical construct;
- two to four supporting evidence blocks;
- one implication or decision box;
- concise source notes.

Split a page when it contains multiple unrelated conclusions. Combine pages when evidence is repetitive and does not advance the argument.

## Recommended outline output

Before production, present:

```text
Overall SCR
Page 1 — conclusion / role / evidence / visual / bridge
Page 2 — conclusion / role / evidence / visual / bridge
...
Open data conflicts
Decisions required from the user
```

Stop and request the first confirmation.

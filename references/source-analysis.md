# Source Analysis

## Purpose

Convert heterogeneous source files into a traceable evidence base before writing slides.

## Required evidence table

Use one row per claim, number, comparison, or recommendation:

| ID | Claim or data | Value | Unit | Period | Source location | Confidence | Conflict or caveat |
|---|---|---:|---|---|---|---|---|

Record exact page, sheet, section, paragraph, table, or filename where possible. Keep original units alongside normalized units.

## Workflow

1. Inventory every supplied file and identify its role.
2. Extract headings, narrative claims, tables, figures, dates, units, and source notes.
3. Separate facts from interpretation and recommendations.
4. Normalize comparable units and periods without overwriting original values.
5. Calculate derived metrics only when inputs and formulas are explicit.
6. Mark missing evidence instead of filling gaps with general knowledge.
7. Reconcile duplicate claims and flag contradictions.
8. Identify evidence that is strong enough for executive conclusions.

## Data conflict protocol

When source statements disagree:

1. Preserve both values.
2. Show each source location and period.
3. Recalculate only when the underlying inputs are present.
4. Explain the likely difference: rounding, period, scope, denominator, forecast versus actual, or source error.
5. Recommend a presentation treatment.
6. Ask the user to choose when the difference changes the conclusion.

Never silently “fix” a source number.

## Missing information

Use explicit labels:

- `Not provided`
- `Cannot be derived`
- `Directional only`
- `External verification required`

Do not fabricate market sizes, growth rates, rankings, survey results, quotations, or sources. If current external research is required, obtain permission or follow the available research workflow and cite it separately.

## ImageGen boundary

Generated images can establish composition, mood, illustration, and visual language. Never use generated text, numbers, logos, citations, or chart values as evidence. Rebuild all factual content from the evidence table.

## Exit criteria

Do not begin SCR design until:

- key claims have source locations;
- units and periods are visible;
- material conflicts are recorded;
- missing critical evidence is disclosed;
- the strongest evidence for each likely conclusion is identifiable.
